## Neo4j 規則

  buildingSize / 10000 * 100) / 100（如果有車位一定要詢問車位價格！非常重要）

### 各平台貼文規則

**FB（Facebook）**
- 長度：300–500 字
- 結構：開場情境 → 物件重點 → 生活機能 → 揭露聲明 → CTA
- 可用條列，適合詳細說明格局與生活機能
- 揭露聲明放在本文末段（非留言）
- CTA：邀請私訊或加 LINE 詢問

**IG（Instagram）**
- 長度：150–250 字
- 結構：吸睛第一句 → 核心亮點 2–3 點 → 揭露聲明 → hashtag
- 簡潔有畫面感，適合用換行製造呼吸感
- Hashtag 10–15 個，含地區、格局、生活關鍵字
- 揭露聲明放在 hashtag 之前的本文

**Threads**
- 長度：100–180 字
- 風格：個人觀點出發，像在跟朋友說「我今天遇到一間房子⋯⋯」
- 不像廣告，更像心得分享
- 揭露聲明放在本文（不放留言）
- 不需要 hashtag
